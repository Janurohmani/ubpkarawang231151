Pengenalan SQL dan Kategorinya

Memahami SQL, DDL, DML, DCL, dan TCL

1. Apa Itu SQL?

SQL (Structured Query Language) adalah bahasa standar yang digunakan untuk mengelola dan memanipulasi basis data relasional. SQL digunakan untuk melakukan berbagai operasi pada data, termasuk pengambilan, penyimpanan, pembaruan, dan penghapusan data.

2. DDL (Data Definition Language)

DDL adalah subset dari SQL yang digunakan untuk mendefinisikan struktur basis data. Perintah DDL digunakan untuk membuat, mengubah, dan menghapus objek basis data seperti tabel, indeks, dan skema.

Contoh Perintah DDL:
-- Membuat tabel
CREATE TABLE siswa (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nama VARCHAR(100),
    umur INT,
    kelas VARCHAR(10)
);

-- Mengubah tabel
ALTER TABLE siswa ADD COLUMN alamat VARCHAR(255);

-- Menghapus tabel
DROP TABLE siswa;

3. DML (Data Manipulation Language)

DML adalah subset dari SQL yang digunakan untuk memanipulasi data dalam tabel. Perintah DML digunakan untuk menambah, mengubah, dan menghapus data dalam basis data.

Contoh Perintah DML:
-- Menambahkan data
INSERT INTO siswa (nama, umur, kelas) VALUES ('John Doe', 15, '10A');

-- Mengubah data
UPDATE siswa SET umur = 16 WHERE nama = 'John Doe';

-- Menghapus data
DELETE FROM siswa WHERE nama = 'John Doe';

4. DCL (Data Control Language)

DCL adalah subset dari SQL yang digunakan untuk mengontrol akses ke data dalam basis data. Perintah DCL digunakan untuk memberikan atau mencabut hak akses pengguna.

Contoh Perintah DCL:
-- Memberikan hak akses
GRANT SELECT, INSERT ON siswa TO 'user'@'localhost';

-- Mencabut hak akses
REVOKE INSERT ON siswa FROM 'user'@'localhost';

5. TCL (Transaction Control Language)

TCL adalah subset dari SQL yang digunakan untuk mengelola transaksi dalam basis data. Perintah TCL digunakan untuk mengontrol transaksi, termasuk menyimpan dan membatalkan perubahan.

Contoh Perintah TCL:
-- Memulai transaksi
START TRANSACTION;

-- Menyimpan perubahan
COMMIT;

-- Membatalkan perubahan
ROLLBACK;

Kesimpulan

SQL adalah bahasa yang sangat penting dalam pengelolaan basis data relasional. Memahami kategori-kategori SQL seperti DDL, DML, DCL, dan TCL sangat penting untuk melakukan operasi yang efektif dan efisien pada basis data.
